## 为什么你的 Obsidian 主题不仅仅是视觉糖果 {#why-theme-matters}

视觉环境与持续性高强度脑力工作的质量之间存在着已被充分证明的联系。Don Norman 在《设计心理学》(The Design of Everyday Things) 中对此进行了深入探讨——其核心论点是，设计精良的物品会让你有使用的欲望，而设计糟糕的工具则会产生摩擦力，并在时间的推移中不断累积。同样的逻辑也适用于软件。

如果你每天打开 Obsidian，而默认的界面让你毫无感觉——没有期待，没有舒适感，也没有一丝愉悦——那你就在白白浪费动力。这对于[笔记记录](/zh-cn/posts/streamlining-your-daily-note-workflow-for-better-productivity/)来说尤为重要，因为它是一种习惯，而不是一项任务。习惯的生死取决于它启动的难易程度，而视觉吸引力是切实有效的切入点。

这其中也有实用层面的考量。对比度差的主题会在 45 分钟后让你的眼睛感到疲劳。层级混乱的主题会让你在浏览自己笔记时感到吃力。排版选择会影响阅读速度。这些不仅仅是审美偏好；它们是可用性变量，并且会在你每一次使用中累积。

从 Obsidian 的默认外观转向精心挑选的主题，其目的并不在于装饰。它是为了将一个功能性的工具转化为一个你真正愿意花时间沉浸其中的工作空间。把它想象成日光灯办公室和设计精良的工作室之间的区别——即使技术上做的是同样的工作，但后者能持续产出更高质量的成果。

## 我们如何评判“美”：我们的评选标准 {#curation-criteria}

本文中的每个主题都是根据四个具体标准进行评估的。这并非是将社区浏览器中的所有主题倾倒出来。

**美学一致性**
是否每一个元素——侧边栏、编辑区、标题、代码块、标签、关系图谱 (graph view)——都感觉属于同一个设计系统？如果在编辑器里看起来很精致，但在设置页面或关系图谱中却显得混乱，这样的主题将落选。

**高强度使用下的可读性**
每个主题都在适用的情况下（亮色或深色模式），通过 90 分钟的写作环节进行了测试。对比度对照了 WCAG AA 标准。无论截图看起来多么漂亮，只要会让眼睛感到不适，该主题就不合格。

**Style Settings 兼容性**
Style Settings 是一款社区插件，它在 Obsidian 内部提供了一个设置面板。你可以在这里更改字体、切换功能、调整强调色等——而无需修改任何 CSS 代码。能与 Style Settings 深度集成的主题得分更高，因为它们赋予你控制权，且不需要你具备编程知识。

**活跃的维护状态**
Obsidian 定期更新其 API。一个最后更新于 2021 年的主题肯定会出问题。在撰写本文时，入选此列表的每个主题在过去八个月内都在 GitHub 上有代码提交。

## 类别一：带来极致专注的极简与整洁主题 {#minimalist-themes}

极简主题移除了所有不直接服务于阅读或写作的元素。没有渐变，没有装饰性元素，减少了 UI 边框。如果你的工作包含长时间的写作——无论你是小说家、研究员还是日常写日记的人——这些主题将最大程度地提升你的[生产力](/zh-cn/posts/obsidian-vs-reflect-for-fast-daily-journaling/)。

### Things

**Things** 是由 Colin Hou 开发的，其设计参考非常明确：它希望 Obsidian 感觉像是一个原生的 macOS 应用程序。侧边栏使用了柔和的灰色，活动笔记的指示器是干净的蓝色，整体印象更接近于 Bear 或 Apple Notes，而不是一个开发者工具。

Things 的出彩之处在于其对标题的处理。H1 到 H3 进行了等比例缩放，并带有合理的字重差异，因此笔记结构立刻变得清晰易读，而不会显得标题喧宾夺主。代码块采用了浅色背景的干净样式，不会打断阅读的心流。

**最佳搭配：**
- **Style Settings**: 在棕褐色、纯白和冷灰等基础颜色之间切换
- **Banners**: 添加全宽的顶部图片，完美融入 Things 的干净布局中
- **[Minimal Theme](/zh-cn/posts/things-theme-vs-minimal-theme-obsidian/)** 通过 Style Settings 的开关提供同等程度的定制化

**适合人群：** macOS 用户、作家、从 Apple Notes 或 Bear 迁移过来的用户

---

### Minimal

如果你只打算安装一个主题，那么 Kepano 开发的 **Minimal** 无疑是最稳妥的选择。自 2020 年以来，它一直保持着持续维护，它自带一个专属的配套插件 (Minimal Theme Settings)，并且它的设计理念令人信服：剥离所有非必要的元素，然后让 Style Settings 和配套插件允许你精确添加你想要的东西。

它的默认状态几乎是极其朴素的——而这正是其目的所在。一旦你安装了配套插件和 Style Settings，你就能掌控卡片布局、图片网格、复选框样式、专注模式、行宽、标题大小以及表格样式。在免费主题中，这种深度的定制化是无与伦比的。

Minimal 的关系图谱也非常出色：节点颜色会根据你选择的强调色进行调整，而不是固定不变的，这听起来是件小事，但看到效果后你就会感叹它的精妙。

**最佳搭配：**
- **Minimal Theme Settings 插件**（必备的配套插件——强烈建议立刻安装）
- **[Dataview](/zh-cn/posts/creating-automated-index-pages-with-obsidian-dataview/)**: Minimal 的卡片和表格样式让 Dataview 的输出变得极具吸引力
- **Kanban**: 看板在 Minimal 的卡片布局模式下渲染得非常清晰

**适合人群：** 进阶用户、[开发者](/zh-cn/posts/best-obsidian-plugins-for-developers-and-code-snippets/)、任何追求长期灵活性的用户

---

### Typora-Vesper

**Vesper** 虽然不如前两者出名，但社区正在逐渐认识到它的价值。它从代码编辑器中流行的 Vesper 配色方案汲取设计灵感——深炭灰背景搭配温暖的琥珀色和奶油色点缀——并将其应用于无干扰的写作界面中。

其核心区别在于它的 Markdown 渲染理念。在实时预览模式下，Vesper 比其他主题更激进地隐藏了 Markdown 语法，因此星号、括号和井号在你写作时几乎完全消失了。屏幕上呈现的是干净的文字。当光标移至标题行时，你能看到结构标记；移开后，这些标记就会隐藏。这切实地改变了在 Obsidian 中写作的体验。

**最佳搭配：**
- **Typewriter Scroll 插件**: 保持当前行居中——与 Vesper 的专注写作体验完美契合
- **Word Count**: Vesper 的状态栏区域能够很好地容纳该插件的输出，且不会显得视觉杂乱

**适合人群：** 作家、记者、从事长篇散文创作的人

## 类别二：舒适精致的深色模式主题 {#dark-mode-themes}

这些主题专为那些长期使用深色模式，并希望获得比简单的反色方案更温暖、更精致体验的人而设计。它们注重夜间工作时的眼睛舒适度、丰富的配色方案以及整体上的视觉工艺感。

### Nord

**Nord** 基于 Nord 配色方案——这是一套源自北极灵感的冷色调蓝色、柔和的青色和白色，最初是为代码编辑器开发的。它很早就被引入到 Obsidian 主题中，并积累了社区所有主题中最庞大的用户群之一。

其吸引力在于一致性。如果你已经在终端、代码编辑器或系统主题中使用了 Nord，你的 Obsidian 库将会感觉与你的整个工作环境融为一体。其蓝色调的饱和度被降低得恰到好处，避免了视觉疲劳；深色背景是真正的深海军蓝，而不是纯黑色；文本的对比度也能在长时间使用中保持舒适。

Nord 在 Obsidian 中对代码块的渲染也极其出色，这使其成为保留技术笔记的开发者的自然之选。

**最佳搭配：**
- **Style Settings**: 在较深和较浅的 Nord 变体之间切换
- **Excalidraw**: 当你使用匹配的笔触颜色时，Nord 的调色板会延伸到 Excalidraw 的草图中
- **Code Block Copy**: Nord 的代码块阅读体验极佳——复制按钮插件更是锦上添花

**适合人群：** 开发者、工程师、系统主题一致性的狂热爱好者

---

### Sanctum

**Sanctum** 处于深色模式色谱上与 Nord 相对的另一端。Nord 显得冷静且实用，而 Sanctum 则显得温暖、紧凑，并且被刻意设计得充满内涵。它使用了深褐灰色背景而不是蓝黑色，并搭配柔和的金色和奶油色用于标题，赋予整个笔记库一种更接近于精心设计的出版物的感觉，而不是一个软件界面。

Sanctum 在排版的选择上尤为出彩。默认的标题栈创建了清晰的视觉层级，正文的字重经过精心调整以提升可读性。侧边栏使用了微妙的分割线而不是粗重的边框，保持了界面的统一感。

如果你希望你的个人[知识管理](/zh-cn/posts/using-obsidian-for-long-term-evergreen-note-management/)库感觉像一个真正的*图书馆*——经过深思熟虑的、从容不迫的、严肃的——Sanctum 是所有免费 Obsidian 主题中最接近这种感觉的。

**最佳搭配：**
- **Banners**: 在 Sanctum 中，顶部的图片感觉更像是精心策展的画作，而非简单的装饰
- **Calendar 插件**: Sanctum 的侧边栏处理让日历小部件看起来非常精致
- **Sliding Panes**: 在 Sanctum 的间距设置下，多面板布局显得很有条理，而不会让人觉得杂乱

**适合人群：** 学者、[研究员](/zh-cn/posts/obsidian-vs-heptabase-for-visual-research-workflows/)、[Zettelkasten](/zh-cn/posts/linking-your-notes-for-better-knowledge-discovery-obsidian/) 实践者、构建长期知识库的写作者

---

### Everforest

**Everforest** 汲取了 Sanctum 的温暖，并融入了饱和的绿色。基于 sainnhe 设计的 Everforest 配色方案，这个 Obsidian 移植版为你的笔记库带来了深邃的森林绿、大地棕和柔和的黄色。其效果是真的舒适——视觉上就如同坐在窗边，看着外面的树木工作一般。

这种暖色调的深色背景（完全避开了冷灰色），使其成为在夜间使用时对眼睛最友好的主题之一。强调色经过了极其细致的平衡，确保代码块、标签和 callout 块都能保持清晰易读，而没有任何元素显得刺眼。

**最佳搭配：**
- **Style Settings**: 在硬朗、中等和柔和的背景变体之间切换
- **Dataview**: 在 Everforest 下，表格的样式非常整洁，并有效地使用了绿色强调色
- **Tasks 插件**: 复选框和任务指示器使用了 Everforest 的调色板，不会破坏视觉上的宁静

**适合人群：** 喜欢在夜间记笔记的人、自然美学爱好者、认为冷色调深色模式过于刻板的人

## 类别三：激发灵感的活力独特主题 {#vibrant-themes}

这些主题并不试图隐藏自己。它们拥有个性、色彩和视觉上的野心。如果你希望你的笔记库感觉像是一个创意的空间，而不是一个生产力工具，这就是你的首选类别。

### Primary

Cecilia May 创作的 **Primary** 是整个 Obsidian 社区中最具原创性的主题之一。其设计语言显然受到了包豪斯 (Bauhaus) 风格的启发：原色、粗边框、几何形状，以及一种被刻意设计的扁平感，让人觉得开朗而不是幼稚。笔记库的 UI 感觉更像是一本设计精美的实体手帐，而不是一个软件应用程序。

Primary 中的标题采用了大胆且带有色块的设计。复选框使用了粗线条。侧边栏具有清晰的视觉结构。它毫不掩饰其张扬的视觉表现力。如果你觉得极简主义让人感到匮乏，那么 Primary 绝对会让你感到耳目一新。

**最佳搭配：**
- **Kanban**: Primary 设计系统中的卡片边框让看板看起来就像是专为此主题量身定制的
- **Banners**: 明亮的顶部图片与 Primary 充满自信的色彩相得益彰，不会产生冲突
- **Colored Tags 插件**: Primary 的标签样式加上自定义的标签颜色，创造了一个极其有趣的组织系统

**适合人群：** 创意专业人士、学生、子弹笔记记录者、认为单色工具令人缺乏动力的人

---

### AnuPpuccin

**AnuPpuccin** 基于 Catppuccin 配色方案——包含四种柔和粉彩风味（Latte, Frappé, Macchiato, Mocha），在开发者工具中大受欢迎。由 AnubisNekhet 开发的 Obsidian 移植版通过大量的 UI 自定义扩展了基础调色板，并且是截至 2024 年下载量最高的社区主题之一。

让 AnuPpuccin 有别于直接照搬配色的原因，在于它附带的自定义 CSS 代码：备用复选框、彩虹文件夹颜色、自定义的 callout 块样式，以及可以通过 Style Settings 访问的动画元素。你可以花一个小时来定制 AnuPpuccin，最终得到一个与别人完全不同的笔记库，哪怕你们使用的是同一个基础主题。

柔和的粉彩色调也意味着它在浅色 (Latte) 和深色 (Mocha, Macchiato) 模式下都能表现出色，而不会让人觉得是两套不同的主题。

**最佳搭配：**
- **Style Settings**（必不可少——AnuPpuccin 的自定义面板选项极其丰富）
- **Iconize**: 以 Catppuccin 强调色渲染的自定义文件夹图标在视觉上非常统一，且一目了然
- **Hider 插件**: 隐藏你不需要的 UI 元素，使粉彩美学更加纯粹

**适合人群：** 注重设计的用户、学生、已经在其他工具中使用 Catppuccin 的人

---

### Prism

damiankorcz 开发的 **Prism** 独树一帜。它的标志性特征是对颜色的处理方式：不是固守单一调色板，而是提供了一个可以通过 Style Settings 访问的庞大预设配色方案库，同时支持将自定义的渐变强调色应用于标题和 UI 元素。

它的技术实现令人印象深刻——特别是应用于 H1-H3 标题的渐变效果，使得结构化的笔记呈现出一种没有其他免费主题能做到的社论版式感。你无需离开 Obsidian 的设置页面，就可以在炫酷的赛博朋克调色板和温暖的日落调色板之间切换。Prism 的关系图谱也极其出色，它使用活动的强调色对节点连接进行了颜色编码。

但这也带来了妥协：因为 Prism 提供了太多选择，你需要花更长的时间去配置才能达到你满意的稳定状态。建议预留 20 到 30 分钟的时间来进行设置。

**最佳搭配：**
- **Style Settings**（必须安装，才能访问 Prism 完整的配色库）
- **Graph Analysis 插件**: Prism 的关系图谱非常出色，花时间在其中是绝对值得的
- **Obsidian Charts**: Prism 的颜色系统能够非常有效地应用到图表渲染中

## 快速指南：如何安装与切换主题 {#how-to-install}

安装社区主题只需要不到两分钟的时间。以下是具体步骤：

**步骤 1：打开外观设置**
前往 `Settings → Appearance`。向下滚动至“Themes”部分，然后点击“Manage”或“Browse community themes”。

**步骤 2：搜索并安装**
Obsidian 内部将打开社区主题浏览器。通过名称进行搜索（例如“Minimal”或“AnuPpuccin”），点击该主题，然后点击“Install and use”。该主题将立即生效。

**步骤 3：在主题之间切换**
返回 `Settings → Appearance → Themes`。下拉菜单会显示所有已安装的主题。你可以随时切换，这不会导致笔记或设置丢失。

**专家提示：立刻安装 Style Settings**
前往 `Settings → Community Plugins → Browse`，搜索“Style Settings”，安装并启用它。然后返回 `Settings → Style Settings`——如果你当前的主题支持它，就会出现一个完整的定制化面板。你可以在这里更改字体、切换 UI 元素、调整强调色等，一切操作都无需代码。

**想要突破 Style Settings 的限制吗？** 基本的 CSS 技能将赋予你对 Obsidian 中每个视觉元素的完全控制权。一个很实用的起点是 Skillshare 上的这门写给[初学者](/zh-cn/posts/obsidian-vault-structure-digital-gardening-beginners/)的 CSS 课程，它可以让你在几个小时内从零基础到能够编写实用的代码片段。Obsidian 可以在 `Settings → Appearance → CSS Snippets` 下原生支持 CSS 代码片段。

## 超越主题：用这些插件完善你的美学 {#aesthetic-plugins}

主题只是打好地基。下面这些插件将进一步扩展它。

### Banners
Banners 插件可以在任何笔记的顶部添加一张全宽的图片。只需添加一个简单的 YAML 属性 (`banner: "path/to/image.jpg"`)，笔记打开时就会显示带有标题图片。在 Sanctum、Minimal 或 Things 主题下，这种效果是非常显著的——笔记不再像是一份文档，而更像是一个精美的页面。它与 Unsplash 的图片或你自己的摄影作品非常搭配。

### Iconize（自定义图标）
Iconize 允许你为侧边栏中的个别文件、文件夹或文件类型分配图标。其实际效果非常惊艳：只需扫一眼带有精心挑选图标的文件夹，甚至不需要阅读文字就能进行导航。在 AnuPpuccin 或 Primary 主题下，使用强调色调色板渲染的图标会让侧边栏感觉像是专门设计的，而不是千篇一律的。

### Supercharged Links
Supercharged Links 为你笔记库中的所有内部链接添加了可视化的[元数据](/zh-cn/posts/explaining-obsidian-properties-for-advanced-metadata-schemas/)。你可以配置它根据链接的属性，在链接上显示彩色圆点、emoji 或者是前缀文本——因此，指向一个被标记为 `#book` 的笔记链接会显示一个书本的 emoji，指向 `#person` 的链接会显示像人名样式的前缀。在任何充满活力的主题下，这都能创造出一个极具视觉效果的链接系统。

### 通过 Envato Elements 使用自定义字体
如果你想将你笔记库的排版提升到一个免费主题包无法提供的水平，Envato Elements 订阅里包含了数千种高级字体和图标集。通过 Obsidian 的 `Settings → Appearance → Text Font` 字段加载你下载的字体（在系统范围内安装字体后指定字体系列名称）。将 Minimal 或 Sanctum 与高级衬线正文字体搭配使用，将显著改变你的阅读体验。

## 结论 {#conclusion}

上述三个类别中的九个主题涵盖了实践中打造优美 Obsidian 外观的现实全景。总结一下实用的决策建议：

- 如果你想将注意力从界面上移开，更多地专注于写作：选择 **Minimal** 或 **Things**

## 常见问题

### 安装主题会以任何方式影响我的笔记或笔记库数据吗？

不会。主题纯粹是 CSS——它们只会改变你笔记库的外观，仅此而已。你的笔记、链接、附件和插件数据与主题文件是完全独立的。你可以随意安装、切换和卸载主题，这对你的内容没有任何风险。

### Obsidian 更新后，我选择的主题看起来坏掉了。我该怎么办？

当 Obsidian 的 API 发生变化，并且主题 CSS 引用了旧的类名时，就会出现这种情况。请查看该主题的 GitHub 仓库，寻找最近的提交或未解决的问题。对于这个列表中的主题，开发者们都很活跃，通常会在 Obsidian 更新后的几天内推送修复程序。在此期间，你可以先切换到一个备用主题（Minimal 是一个安全的后备选择——它的更新非常迅速）。

### 我可以同时使用多个主题，或者混合使用两个主题的元素吗？

不能直接实现。一次只能激活一个主题。但是，你可以使用 CSS 代码片段 (`Settings → Appearance → CSS Snippets`) 来覆盖当前活动主题的特定元素。如果你熟悉基础的 CSS，你就可以保留当前主题配色的同时，借用另一个主题的标题样式。

### 哪个主题最适合阅读障碍者或有阅读困难的人？

Minimal 和 Things 是最理想的起点，因为它们的排版非常干净，而且行间距很大。无论在哪个主题中，你都可以通过 Style Settings 增加行高和字体大小。Everforest 和 Sanctum 的暖色调背景也能降低纯白或纯黑背景带来的刺眼感，一些对阅读敏感的用户发现这很有帮助。

### 这些主题支持 Obsidian 移动端 (iOS/Android) 吗？

是的，这里列出的所有主题都支持移动端。体验质量可能略有差异——某些 Style Settings 选项仅限桌面端使用。但核心的视觉体验（颜色、排版、布局）都能正确地转移到移动端上。如果你经常在手机上使用你的笔记库，Minimal 的配套插件有一些专门针对移动端布局的优化选项，非常值得开启。
